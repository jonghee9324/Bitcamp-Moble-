@java 클래스 변수 = static
같은 클래스로 만들어진 객체들이 모두 공유하는 변수
객체 생성없이 클래스이름.클래스 변수명 사용가능

변수명 앞에 static이 붙으면 클래스변수됨
클래스 변수로 만드는 방법은 static뿐

@@static
static이 붙으면 메모리 공간이 달라짐, 인스턴스변수와 다른위치의 메모리에 클래스변수가 저장.
static이 붙은 변수는 다른 객체에서 값을 변경해도 같은 클래스의 객체는 해당 변수의 값이 변경된다.
static이 붙은 메소드는 static이 붙은 메소드, 변수만 호출이 가능하다.


@메소드
if else문에서 각각 리턴하는 것은 좋은 코드가 아님
1번만 리턴할 것

@@ 오버로딩
이름이 같고 매개변수가 다른 함수를 같은 함수로 인식하는 것 

@생성자
반환타입이 없는 클래스 이름과 동일한 메소드
기본 생성자는 안에 내용이 없다. 생략됨
생성자에서 생성자 호출가능(단, 생성자에서 생성자 호출 시 이 때는 생성자 대신 this를 사용)
생성자의 매개변수에 다른 인스턴스가 들어올 수 있음
