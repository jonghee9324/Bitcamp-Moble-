int 보다 크기가 작은 타입은 int로 바꾼다.
즉, short, byte를 계산 시 int로 바꿔 결과가 int로 나온다. 따라서 두 개의 자료형으로 연산한 결과는 각 자료형에 넣지못함
연산시 표현범위가 다르면, 표현범위가 큰 데이터로 결과값이 나온다.
ex) long * int = long
int * int의 결과가 표현범위를 넘어가면 long으로 사용해야함

정수, 실수뒤에 접미사가 없는경우는 각 데이터의 기본 값인 int, double로 취급된다.

자료형은 데이터를 표시하는 단위이고, 표현할 수 있는 범위가 존재한다. 그 범위는 바이트 크기에 비례한다.
4byte = 32비트 = 2^32까지 표현가능

자바 디버깅은 문번호 왼쪽 브레이크 포인트를 걸 곳을 더블클릭 화살표 왼쪽 벌레 모양을 누르면 디버깅모드
디버깅 모드 시 변수에 마우스를 가져가면 값이 보임
변수 오른쪽 클릭 watching을 누르면 watching 창이 열려 변수의 변화를 볼 수 있다.
다른 변수를 확인하고 싶으면 와칭창 add로 변수를 추가하거나
변수 오른쪽클릭 watching을 통해 변수를 추가할 수있다.

디버깅 모드에서 한줄씩 이동은 f6, 메소드 내부로 들어갈 땐 f5, 중지는 빨간색 네모(stop)

정수 문자의 정수화 : 아스키코드표의 0값인 48을 빼준다.
정수의 자릿수 찾기 : (int)(Math.log10(a)+1);
